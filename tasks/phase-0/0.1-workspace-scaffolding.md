# Task 0.1: Workspace Scaffolding

## Objective
Create the Cargo workspace with all five crates, project root files, and verify everything compiles.

## Steps
1. Initialize the Cargo workspace at the project root with `crates/*` as members.
2. Create the five crates:
   - `crates/api` — binary crate (`main.rs`)
   - `crates/domain` — library crate (`lib.rs`)
   - `crates/db` — library crate (`lib.rs`)
   - `crates/importer` — library crate (`lib.rs`)
   - `crates/price-fetcher` — library crate (`lib.rs`)
3. Set up inter-crate dependencies in each `Cargo.toml`:
   - `api` depends on `domain`, `db`
   - `db` depends on `domain`
   - `importer` depends on `domain`
   - `price-fetcher` depends on `domain`
4. Add `//!` crate-level doc comment in each `lib.rs` and `main.rs`.
5. Create root files:
   - `.gitattributes` (LF endings, xlsx binary)
   - `.gitignore` (target/, .env, *.swp, .DS_Store)
   - `.env.example` with all config parameters and placeholder values
   - `docs/` directory with `makima-mvp-spec.md` (copy from existing)
6. `main.rs` in `api` should contain a minimal `fn main()` with a `println!("Makima starting...")`.

## Done criteria
- `cargo build` succeeds with zero errors
- `cargo clippy -- -D warnings` passes
- `cargo fmt --check` passes
- `cargo doc --no-deps` builds without warnings
- Project structure matches the workspace layout in CLAUDE.md
