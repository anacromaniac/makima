# Task 0.4: OpenAPI and Swagger UI

## Objective
Set up automatic OpenAPI 3.0 spec generation with utoipa and serve Swagger UI from the backend.

## Steps
1. Add `utoipa` and `utoipa-swagger-ui` dependencies to `crates/api/Cargo.toml`. Use the axum integration feature.
2. Create the OpenAPI spec struct with:
   - Title: "Makima API"
   - Version: "0.1.0"
   - Description: "Investment tracking backend API"
   - API prefix: `/api/v1`
3. Annotate the /health and /ready endpoints with utoipa macros (`#[utoipa::path(...)]`).
4. Serve Swagger UI at `/swagger-ui`.
5. Serve the raw OpenAPI JSON spec at `/api-doc/openapi.json`.

## Done criteria
- `curl http://localhost:3000/api-doc/openapi.json` returns a valid OpenAPI 3.0 JSON spec.
- Opening `http://localhost:3000/swagger-ui` in a browser shows the Swagger UI with /health and /ready documented.
- The /health and /ready endpoints can be tested directly from Swagger UI.
- All clippy/fmt/doc checks pass.

## Files created/modified
- `crates/api/Cargo.toml` (add utoipa, utoipa-swagger-ui)
- `crates/api/src/main.rs` (OpenAPI setup, Swagger UI route)
