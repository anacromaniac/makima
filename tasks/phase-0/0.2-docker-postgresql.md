# Task 0.2: Docker and PostgreSQL

## Objective
Create the Docker multi-stage build and docker-compose setup with PostgreSQL. The backend container should build and start, connecting to the database.

## Steps
1. Create `Dockerfile` with multi-stage build:
   - Builder stage: Rust compiler image, compile with musl target for Alpine compatibility.
   - Runtime stage: `alpine:latest`, copy the binary only.
2. Create `docker-compose.yml` with two services:
   - `postgres`: PostgreSQL image, volume for data persistence, healthcheck, tuned for RPi 3 (max_connections=20, shared_buffers=128MB, work_mem=4MB).
   - `makima-api`: builds from Dockerfile, depends on postgres (condition: service_healthy), loads `.env` via env_file.
3. Update `crates/api/src/main.rs` to:
   - Load configuration from environment variables using `config` + `dotenvy`.
   - Create sqlx PgPool with configurable max connections (default: 5).
   - Run embedded migrations via `sqlx::migrate!()`.
   - Print "Makima connected to database" on success.
   - Exit with error if database connection or migration fails.
4. Add sqlx, config, dotenvy dependencies to relevant `Cargo.toml` files. Use `runtime-tokio-rustls` feature for sqlx.
5. Create first empty migration: `migrations/0001_initial.sql` (can be just a comment).

## Done criteria
- `docker-compose up --build` starts both containers without errors.
- Backend connects to PostgreSQL and runs the empty migration.
- `docker-compose down -v` cleans up.
- `cargo build` still succeeds outside Docker for local development.
- All clippy/fmt checks pass.
