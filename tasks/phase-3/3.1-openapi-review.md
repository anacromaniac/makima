# Task 3.1: OpenAPI Annotations Review

## Objective
Review and complete utoipa annotations across all endpoints, ensuring Swagger UI is a fully accurate and usable API reference.

## Steps
1. Review all endpoints and ensure every one has complete utoipa annotations:
   - Request body schemas with examples.
   - Response schemas for all status codes (200, 400, 401, 404, 500).
   - Path parameters and query parameters documented.
   - Security requirement (Bearer JWT) on all protected endpoints.
2. Ensure all DTOs (request and response) derive `utoipa::ToSchema`.
3. Ensure all enums (AssetClass, TransactionType) are documented in the OpenAPI spec.
4. Add global error response schema for the standard `{ "code", "message" }` format.
5. Verify the paginated response wrapper `{ "data", "pagination" }` is correctly represented.
6. Test: open Swagger UI and verify every endpoint is documented, tryable, and shows correct schemas.

## Done criteria
- Every endpoint appears in Swagger UI with complete documentation.
- Request/response examples are shown for each endpoint.
- Error responses are documented for all endpoints.
- Auth endpoints show the lock icon and token entry works.
- All clippy/fmt/doc checks pass.
