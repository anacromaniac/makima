# Task 1.2: Traits and Domain Logic

## Objective
Define the BrokerImporter trait and implement core domain logic functions with unit tests.

## Steps
1. Define `BrokerImporter` trait in `crates/domain/src/traits.rs`:
   - Method: takes raw bytes (file content) and returns `Result<Vec<NewTransaction>, DomainError>`.
   - Each broker parser will implement this trait in the `importer` crate.
2. Create `crates/domain/src/logic/` module with:
   - `position.rs` — function to aggregate a list of transactions into a position (held quantity, average cost). Must reject if resulting quantity would be negative (no short selling).
   - `gain_loss.rs` — function to calculate gain/loss for a position given current price. Returns absolute and percentage.
   - `allocation.rs` — function to calculate asset allocation percentages from a list of positions with their current values and asset classes.
3. Write unit tests for all logic functions:
   - Position aggregation: buy only, buy+sell, sell to zero (closed position), sell exceeding quantity (error).
   - Gain/loss: profit case, loss case, zero gain, with different currencies.
   - Asset allocation: single class, multiple classes, empty portfolio.

## Done criteria
- All unit tests pass (`cargo test -p domain`).
- `BrokerImporter` trait is defined and documented.
- Logic functions handle edge cases (empty input, zero quantity, zero price).
- No framework/db dependencies in domain crate.
- All clippy/fmt/doc checks pass.
