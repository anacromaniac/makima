# Task 2.3a: Portfolios Integration Tests

## Objective
Retrofit end-to-end integration tests for portfolio CRUD and ownership isolation using the shared API integration harness.

## Steps
1. Create `crates/api/tests/portfolios.rs`.
2. Cover CRUD behavior:
   - Create portfolio returns `201`.
   - Get owned portfolio returns `200`.
   - Update owned portfolio returns `200`.
   - Delete owned portfolio returns `204`.
3. Cover list and pagination behavior:
   - List returns only the caller's portfolios.
   - Pagination metadata includes `page`, `limit`, `total_items`, and `total_pages`.
   - Default and bounded pagination behavior is asserted.
4. Cover ownership isolation:
   - User A cannot read, update, or delete user B's portfolio.
   - These cases return `404`, not `403`.
5. Cover validation behavior:
   - Invalid create/update payloads return the expected validation error response.

## Done criteria
- All existing portfolio endpoints have request/response integration coverage.
- Ownership isolation is covered end-to-end with two users.
- Pagination and validation behavior are asserted.
- Tests use the shared API integration harness and a dedicated PostgreSQL test database.
- All clippy/fmt/doc/test checks pass.
