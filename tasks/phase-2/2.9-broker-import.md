# Task 2.9: Broker Import (Fineco and BG Saxo)

## Objective
Implement the file upload endpoint and the two broker parsers (Fineco, BG Saxo) using the BrokerImporter trait.

## Steps
1. Implement `BrokerImporter` trait in `crates/importer/src/`:
   - `fineco.rs` — parser for Fineco Excel export format. Implements `BrokerImporter`. Maps Fineco-specific columns to `NewTransaction` fields.
   - `bgsaxo.rs` — parser for BG Saxo Excel export format. Implements `BrokerImporter`. Maps BG Saxo-specific columns to `NewTransaction` fields.
   - Add `calamine` dependency for .xlsx reading.
2. Each parser must:
   - Read the .xlsx file from raw bytes.
   - Map rows to `NewTransaction` structs with correct transaction_type, dates, quantities, prices, commissions, currency, ISIN.
   - Skip header rows, summary rows, or other non-transaction rows.
   - Collect all validation errors per row (row number + error description) without stopping.
   - After parsing: if any row has validation errors, return all errors (no partial result).
3. Create `crates/api/src/import/`:
   - `handlers.rs`:
     - `POST /api/v1/import/{broker}` — accepts multipart file upload. `{broker}` path param selects parser (`fineco` | `bgsaxo`). Requires portfolio_id as query param.
     - Verify portfolio ownership.
     - Parse file with selected broker parser.
     - If parse errors: return 400 with list of row-level errors.
     - If parse succeeds: begin SQL transaction. For each transaction:
       - Look up asset by ISIN. If not found, create via asset_repo + OpenFIGI attempt. If OpenFIGI fails, create with yahoo_ticker = NULL and add to warnings.
       - If transaction currency != EUR, attempt exchange rate fetch. If fetch fails, store exchange_rate_to_base = NULL and add to warnings.
       - Insert transaction.
     - If any INSERT fails: rollback entire SQL transaction, return error.
     - On success: commit, return summary.
   - `dto.rs`:
     - `ImportResponse` — transactions_imported (u64), assets_created (Vec<String> of ISINs), warnings (Vec<String> for assets without yahoo_ticker, transactions without exchange rate, etc.).
     - `ImportError` — row_errors (Vec<{ row: u32, message: String }>).
4. Duplicate detection: compute `import_hash` from `date + ISIN + quantity + unit_price + transaction_type`. Before inserting, check if hash exists in portfolio. Skip duplicates and include in response warnings.
5. Commit fixture .xlsx files for testing:
   - `crates/importer/fixtures/fineco_sample.xlsx`
   - `crates/importer/fixtures/bgsaxo_sample.xlsx`
6. Write unit tests in the `importer` crate for both parsers using the fixture files.
7. Annotate endpoint with utoipa macros.

## Done criteria
- Uploading a valid Fineco .xlsx creates all transactions correctly.
- Uploading a valid BG Saxo .xlsx creates all transactions correctly.
- File with invalid rows returns 400 with all row-level errors listed.
- Assets not in the system are auto-created during import.
- OpenFIGI or exchange rate failures result in warnings, not errors — import still succeeds.
- Duplicate transactions are skipped with a warning.
- The entire import is atomic: partial INSERT failure rolls back everything.
- Unit tests pass for both parsers with fixture files.
- Invalid broker name in path returns 400.
- All clippy/fmt/doc/test checks pass.
