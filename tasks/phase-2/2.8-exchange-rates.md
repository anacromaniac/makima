# Task 2.8: Exchange Rates

## Objective
Implement exchange rate fetching from Yahoo Finance, storage, and integration into the daily price update job.

## Steps
1. Create migration `migrations/0007_exchange_rates.sql`:
   - `exchange_rates` table per schema in MVP spec.
   - UNIQUE constraint on `(from_currency, to_currency, date)`.
2. Create `crates/db/src/repositories/exchange_rate_repo.rs`:
   - `insert(new_rate) -> Result<ExchangeRate>` (upsert on conflict)
   - `find_latest(from_currency, to_currency) -> Result<Option<ExchangeRate>>`
   - `find_by_date(from_currency, to_currency, date) -> Result<Option<ExchangeRate>>`
3. Create `crates/price-fetcher/src/exchange.rs`:
   - `fetch_rate(from_currency, to_currency) -> Result<ExchangeRate>` — fetch from Yahoo Finance using pairs like `EURUSD=X`.
   - Same retry/backoff strategy as price fetching.
4. Integrate into the daily job (`crates/price-fetcher/src/job.rs`):
   - After updating asset prices, determine which currencies are in use across all transactions (all currencies != EUR).
   - Fetch and store the current exchange rate for each currency pair to EUR.
   - Log summary: updated N exchange rates, failed M.
5. Integrate into transaction creation (`crates/api/src/transactions/handlers.rs`):
   - When creating a transaction in a non-EUR currency, look up the latest exchange rate from the DB.
   - If no rate exists, attempt a live fetch from Yahoo Finance.
   - If both fail, require `exchange_rate_to_base` in the request body, or reject with 400.

## Done criteria
- Daily job fetches and stores exchange rates for all currencies in use.
- Transaction creation in non-EUR currency automatically resolves exchange rate.
- Transaction creation with unavailable exchange rate and no manual rate provided returns 400.
- Latest rate lookup works correctly.
- All clippy/fmt/doc/test checks pass.
