# Task 2.10: Analytics (Portfolio Summary and Asset Allocation)

## Objective
Implement the portfolio summary endpoint with gain/loss and asset allocation by class.

## Steps
1. No migration needed — all calculations are on-the-fly.
2. Create `crates/api/src/portfolios/handlers.rs` (add to existing):
   - `GET /api/v1/portfolios/{id}/summary` — verify portfolio ownership. Returns:
     - Total portfolio value in EUR (sum of all position current values, converted via exchange rates).
     - Total gain/loss absolute and percentage.
     - Asset allocation breakdown by asset class.
3. Create `crates/api/src/portfolios/dto.rs` (add to existing):
   - `PortfolioSummaryResponse`:
     - `total_value` (Option<Decimal>) — null if no prices available.
     - `total_gain_loss_absolute` (Option<Decimal>)
     - `total_gain_loss_percentage` (Option<Decimal>)
     - `asset_allocation` (Vec<AssetAllocationEntry>)
     - `warnings` (Vec<String>) — positions missing prices or exchange rates.
   - `AssetAllocationEntry`:
     - `asset_class` (AssetClass)
     - `value` (Decimal)
     - `percentage` (Decimal)
4. Implementation flow:
   - Fetch all open positions for the portfolio (reuse position logic from task 2.6).
   - For each position: get latest price, calculate current value. Convert to EUR using latest exchange rate if needed.
   - Use domain logic functions from task 1.2 for gain/loss and allocation calculation.
   - Positions without current price are excluded from totals with a warning in the response.
5. Annotate endpoint with utoipa macros.

## Done criteria
- Summary returns correct total value, gain/loss, and allocation.
- Multi-currency positions are correctly converted to EUR.
- Positions without prices are excluded from totals and listed in warnings.
- Asset allocation percentages sum to 100%.
- Empty portfolio returns zero values and empty allocation.
- User can only access summaries of their own portfolios.
- All clippy/fmt/doc/test checks pass.
