# Task 2.2: Auth Middleware and User Endpoint

## Objective
Create the JWT authentication middleware that protects all /api/v1/* routes (except /auth/*) and the user profile endpoint.

## Steps
1. Create `crates/api/src/auth/middleware.rs`:
   - Axum extractor `AuthUser` that reads `Authorization: Bearer <token>` header.
   - Verifies JWT signature and expiration.
   - Extracts `user_id` from claims.
   - Returns 401 with `{ "code": "UNAUTHORIZED", "message": "..." }` on failure (missing header, invalid token, expired token).
   - Makes `user_id: Uuid` available to downstream handlers.
2. Create `crates/api/src/users/handlers.rs`:
   - `GET /api/v1/users/me` — requires `AuthUser`, returns user profile (id, email, created_at). Never returns password_hash.
3. Create user response DTO (id, email, created_at only).
4. Apply auth middleware to all `/api/v1/*` routes except `/api/v1/auth/*`.
5. Annotate endpoint with utoipa macros including security scheme (Bearer JWT).

## Done criteria
- `GET /api/v1/users/me` with valid JWT returns user profile.
- `GET /api/v1/users/me` without token returns 401.
- `GET /api/v1/users/me` with expired token returns 401.
- `/health`, `/ready`, and `/api/v1/auth/*` remain accessible without token.
- Swagger UI shows the lock icon and allows entering a Bearer token.
- All clippy/fmt/doc/test checks pass.
