# Task 2.4: Assets CRUD and OpenFIGI Integration

## Status
- [x] Completed on 2026-04-19

## Objective
Implement CRUD for the shared assets table and integrate OpenFIGI for automatic ISIN-to-Yahoo-ticker mapping.

## Steps
1. Create migration `migrations/0004_assets.sql`:
   - `assets` table per schema in MVP spec.
   - Create PostgreSQL enum type for `asset_class`.
   - Unique index on `isin`.
2. Create `crates/db/src/repositories/asset_repo.rs`:
   - `create(new_asset) -> Result<Asset>`
   - `find_by_id(id) -> Result<Option<Asset>>`
   - `find_by_isin(isin) -> Result<Option<Asset>>`
   - `list(pagination, filters) -> Result<PaginatedResult<Asset>>` (filterable by asset_class, name search)
   - `update(id, fields) -> Result<Asset>`
   - `update_yahoo_ticker(id, ticker) -> Result<()>`
3. Create `crates/price-fetcher/src/openfigi.rs`:
   - Client struct that calls OpenFIGI API to map ISIN to Yahoo ticker.
   - Takes optional `OPENFIGI_API_KEY` from env (works without key at lower rate limits).
   - Returns `Option<String>` — the Yahoo ticker, or None if not found.
   - Parse the OpenFIGI response and extract the composite ticker + exchange suffix for Yahoo.
   - Handle errors gracefully: log and return None, never fail the caller.
4. Create `crates/api/src/assets/`:
   - `dto.rs` — CreateAssetRequest (isin required, name required, asset_class required, currency required, exchange optional, garde validated), UpdateAssetRequest, AssetResponse.
   - `handlers.rs`:
     - `GET /api/v1/assets` — list all assets (paginated, filterable). No auth ownership check (assets are shared).
     - `GET /api/v1/assets/{isin}` — get by ISIN.
     - `POST /api/v1/assets` — create asset. If yahoo_ticker not provided, attempt OpenFIGI lookup. If OpenFIGI fails, create with yahoo_ticker = NULL.
     - `PUT /api/v1/assets/{isin}` — update asset fields.
5. ISIN validation: basic format check (2 letter country code + 9 alphanumeric + 1 check digit = 12 chars).
6. Annotate all endpoints with utoipa macros.

## Done criteria
- CRUD operations work correctly.
- Creating an asset without yahoo_ticker triggers OpenFIGI lookup.
- Creating an asset when OpenFIGI is unavailable succeeds with yahoo_ticker = NULL and logs a warning.
- Duplicate ISIN returns appropriate error.
- ISIN format validation rejects invalid formats.
- List endpoint supports filtering by asset_class and name search.
- All clippy/fmt/doc/test checks pass.
