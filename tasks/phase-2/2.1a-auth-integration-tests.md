# Task 2.1a: Auth Integration Tests

## Objective
Retrofit end-to-end integration tests for the existing auth endpoints using the shared API integration harness.

## Steps
1. Create `crates/api/tests/auth.rs`.
2. Cover registration flows:
   - Successful registration returns `201` and a token pair.
   - Duplicate email returns `409`.
   - Invalid payload returns the expected validation error response.
3. Cover login flows:
   - Correct credentials return `200` and a token pair.
   - Wrong password returns `401`.
4. Cover refresh flows:
   - Valid refresh token returns a new token pair and invalidates the old token.
   - Invalid or revoked refresh token returns `401`.
   - Stolen-token detection path revokes all sessions for the user.
5. Cover password change flows:
   - Valid bearer token + correct old password returns `204`.
   - Old password no longer works after change.
   - Existing refresh tokens become unusable after change.

## Done criteria
- All existing auth endpoints have request/response integration coverage.
- Success, validation, authentication failure, and token revocation paths are covered.
- Tests use the shared API integration harness and a dedicated PostgreSQL test database.
- All clippy/fmt/doc/test checks pass.
