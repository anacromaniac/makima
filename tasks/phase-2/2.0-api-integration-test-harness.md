# Task 2.0: API Integration Test Harness

## Objective
Create the shared integration-test infrastructure required to make endpoint-level tests part of each Phase 2 feature task.

## Steps
1. Refactor the `api` crate so tests can build the real Axum app without starting the production binary:
   - Extract router/app construction from `crates/api/src/main.rs` into reusable code.
   - Keep `main.rs` focused on bootstrap, env loading, tracing, migrations, and server startup.
2. Create the integration test directory and shared support modules under `crates/api/tests/`.
3. Implement a shared test harness that:
   - Creates an isolated PostgreSQL test database for each test case or test module.
   - Runs sqlx migrations against that database.
   - Builds the real app with concrete repositories and test configuration.
   - Provides helper methods for common flows such as registration, login, auth headers, and seeded entities.
4. Define the required test environment variables and local workflow for running API integration tests.
5. Ensure the harness is the standard foundation for all future feature integration tests.

## Done criteria
- Integration tests can build the real Axum app without spawning the full binary.
- Each test run uses an isolated PostgreSQL database with migrations applied.
- The harness exposes reusable helpers for authenticated and unauthenticated request flows.
- The repo documents how to run integration tests locally and in CI.
- All clippy/fmt/doc/test checks pass.
