# Task 2.2a: Authenticated User Integration Tests

## Objective
Retrofit end-to-end integration tests for JWT-protected access and the `GET /api/v1/users/me` endpoint.

## Steps
1. Create `crates/api/tests/users.rs`.
2. Cover authentication boundary behavior:
   - `GET /api/v1/users/me` with a valid JWT returns `200`.
   - Missing bearer token returns `401`.
   - Invalid or expired token returns `401`.
3. Verify public-route exclusions:
   - `/health`, `/ready`, and `/api/v1/auth/*` remain accessible without authentication.
4. Verify the response shape for `GET /api/v1/users/me`:
   - Returns `id`, `email`, and `created_at`.
   - Never returns `password_hash`.

## Done criteria
- Protected-route auth behavior is covered end-to-end.
- `GET /api/v1/users/me` is covered for success and failure cases.
- Public-route exclusions are exercised by integration tests.
- Tests use the shared API integration harness and a dedicated PostgreSQL test database.
- All clippy/fmt/doc/test checks pass.
