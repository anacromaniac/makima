# Task 2.1: Auth — Registration, Login, and JWT

## Objective
Implement user registration, login, JWT access token issuance, and refresh token rotation with database persistence.

## Steps
1. Create migration `migrations/0002_users_and_refresh_tokens.sql`:
   - `users` table per schema in MVP spec.
   - `refresh_tokens` table per schema in MVP spec.
   - Index on `users.email` and `refresh_tokens.token_hash`.
2. Create `crates/db/src/repositories/user_repo.rs`:
   - `create(new_user) -> Result<User>`
   - `find_by_email(email) -> Result<Option<User>>`
   - `find_by_id(id) -> Result<Option<User>>`
   - `update_password(id, new_password_hash) -> Result<()>`
3. Create `crates/db/src/repositories/refresh_token_repo.rs`:
   - `create(new_refresh_token) -> Result<RefreshToken>`
   - `find_by_hash(token_hash) -> Result<Option<RefreshToken>>`
   - `revoke(id) -> Result<()>`
   - `revoke_all_for_user(user_id) -> Result<()>`
4. Create auth service logic in `crates/api/src/auth/`:
   - `service.rs`:
     - `register(email, password)` — validate input with garde, hash password with argon2, create user, generate token pair, return tokens.
     - `login(email, password)` — find user, verify password with argon2, generate token pair, return tokens.
     - `refresh(refresh_token_string)` — hash the incoming token, look up in DB, verify not revoked and not expired. If a revoked token is presented, revoke ALL tokens for that user (stolen token detection). On success: revoke old token, generate new pair, return new tokens.
     - `change_password(user_id, old_password, new_password)` — verify old password, update hash, revoke all refresh tokens.
   - `jwt.rs` — JWT creation (HS256, contains user_id and exp) and verification. Access token: 1h. Read `JWT_SECRET` from env.
   - `tokens.rs` — generate cryptographically random opaque refresh token string, SHA-256 hashing function.
5. Create handler endpoints in `crates/api/src/auth/handlers.rs`:
   - `POST /api/v1/auth/register` — request: `{ email, password }`, response: `{ access_token, refresh_token }`.
   - `POST /api/v1/auth/login` — same request/response format.
   - `POST /api/v1/auth/refresh` — request: `{ refresh_token }`, response: `{ access_token, refresh_token }`.
   - `PUT /api/v1/auth/password` — requires valid JWT. Request: `{ old_password, new_password }`.
6. Create request/response DTOs with garde validation:
   - Email: valid email format.
   - Password: minimum 8 characters.
7. Annotate all endpoints with utoipa macros.

## Done criteria
- Register creates a user and returns token pair.
- Login with correct credentials returns token pair.
- Login with wrong credentials returns 401.
- Refresh with valid token returns new token pair and invalidates old.
- Refresh with revoked token revokes ALL tokens for that user.
- Change password works and invalidates all sessions.
- Duplicate email registration returns appropriate error.
- Passwords are never logged (custom Debug on User).
- All clippy/fmt/doc/test checks pass.
