# Task 2.7: Prices, Yahoo Finance Client, Daily Job, and Backfill

## Objective
Implement Yahoo Finance price fetching, price history storage, daily scheduled job, manual refresh endpoint, manual price entry, and historical backfill.

## Steps
1. Create migration `migrations/0006_price_history.sql`:
   - `price_history` table per schema in MVP spec.
   - UNIQUE constraint on `(asset_id, date)`.
   - Index on `asset_id`.
2. Create `crates/db/src/repositories/price_repo.rs`:
   - `insert(new_price_record) -> Result<PriceRecord>` (upsert: on conflict update close_price and source)
   - `insert_batch(records) -> Result<u64>` (bulk upsert for backfill)
   - `find_latest(asset_id) -> Result<Option<PriceRecord>>`
   - `find_by_range(asset_id, from_date, to_date, pagination) -> Result<PaginatedResult<PriceRecord>>`
3. Create `crates/price-fetcher/src/yahoo.rs`:
   - `fetch_current_price(ticker) -> Result<PriceRecord>` — fetch latest price for a single ticker.
   - `fetch_history(ticker, from_date, to_date) -> Result<Vec<PriceRecord>>` — fetch historical daily close prices.
   - Configurable delay between requests (`YAHOO_REQUEST_DELAY_MS`, default 1500ms).
   - Retry with exponential backoff on HTTP 429: 30s, 60s, max 3 retries. Log each retry.
   - Return `ExternalServiceError` on permanent failure, never panic.
4. Create `crates/price-fetcher/src/job.rs`:
   - Scheduled job using tokio-cron-scheduler.
   - Runs at `PRICE_UPDATE_CRON` (default: daily at 22:00).
   - Queries all assets that exist in at least one portfolio.
   - For each asset with a yahoo_ticker: fetch current price, insert into price_history.
   - Batch tickers where Yahoo API supports it.
   - Respect delay between requests.
   - Log summary on completion: updated N assets, failed M.
5. Create backfill logic in `crates/price-fetcher/src/backfill.rs`:
   - `backfill_asset(asset_id, ticker, years) -> Result<u64>` — fetch historical prices for the last N years (`BACKFILL_YEARS`, default 5) and bulk insert.
   - Called when an asset gets a yahoo_ticker for the first time.
6. Create `crates/api/src/assets/handlers.rs` (add to existing):
   - `POST /api/v1/assets/{isin}/refresh-price` — force price update for single asset. Requires yahoo_ticker, returns 400 if NULL.
   - `PUT /api/v1/assets/{isin}/price` — manual price entry. Request: `{ date, close_price, currency }`. Sets source = "manual".
   - `GET /api/v1/assets/{isin}/price-history` — paginated, filterable by date range.
7. Register the scheduled job in `main.rs` startup.
8. Annotate new endpoints with utoipa macros.

## Done criteria
- Manual price entry works and stores with source = "manual".
- Manual refresh fetches from Yahoo Finance and stores with source = "yahoo".
- Refresh for asset without yahoo_ticker returns 400.
- Price history endpoint returns paginated results filterable by date.
- Daily job runs at configured time (testable by triggering manually).
- Backfill populates historical data on first ticker assignment.
- HTTP 429 from Yahoo triggers retry with backoff, logged.
- Job continues on individual asset failure, logs errors.
- All clippy/fmt/doc/test checks pass.
